# Value And Insight Extraction

## Use This File When

- the user has lived experience but no clear观点
- the source is stories, regrets, mistakes, lessons, or fragments
- the content sounds informational but not memorable
- the task is to turn experience into publishable insight

## Core Principle

Information is common.
Transferable insight is scarce.

Do not only repeat what happened.
Extract what another person can use.

## Strong Insight Targets

Prefer insights that do at least one of these:

- break an expectation
- explain why common behavior fails
- show a better sequence
- reveal a hidden rule
- help the viewer avoid a real mistake

## Experience Mining Workflow

Use this four-step pass:

1. painful moment
   Ask for the moment that later felt like "I wish I had known this earlier."
2. three checks
   Test whether the lesson is universal enough, regret-inducing enough, and actionable enough.
3. three layers
   Extract:
   - surface: what happened
   - behavior: what pattern caused it
   - essence: what deeper rule was being violated
4. package the insight
   Turn it into a viewpoint, not just a story.

## Helpful Questions

- What was the scene.
- What decision was made.
- Why did that decision feel reasonable then.
- What did it cost.
- When did the speaker realize it was wrong.
- What should have been done instead.

## Insight Packaging Template

Compress the result like this:

- shock opener
- shared scene
- personal case
- essence explanation
- action guide
- memorable close

## Value Filter

Keep the insight only if it is:

- true to lived experience
- specific in time, scene, or cost
- useful to someone else
- compressible into one line worth saving

Reject it if it is:

- generic advice
- abstract motivation
- vague self-growth talk
- a loose paraphrase with no causal chain

## Useful Output Forms

- one insight seed
- three gold sentences
- one "wrong sequence vs right sequence" contrast
- one lesson card
- one script brief based on the insight
