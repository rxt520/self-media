# Short Video Script System

## Overview

Use this skill to turn vague topics, personal experience, creator goals, or product intentions into positioning-aware Chinese short-video outputs.
Prefer one clear audience, one root problem, one transferable insight, and one next action over isolated hook generation.

## When To Use

- The user wants a Chinese short-video script, hook, title, CTA, or rewrite
- The user needs personal IP positioning before writing content
- The user gives raw experience, regrets, observations, or note dumps and wants usable insights
- The user wants a weekly topic pool, content mainline, or publishing SOP
- The user wants content that supports trust, inquiry, or conversion instead of only play count

Do not use this skill for subtitles, editing, rendering, or post-processing.

If the task is only batch-generating finalized Douyin talking-head scripts, also consider the local companion skill:

- `self-media/skills/script-writing/口播生成复用技能/SKILL.md`

2. value and insight
3. script assembly
4. SOP and conversion
5. connected series workflow

## Standard Workflow

1. Diagnose the current stage.
   Decide whether the real blocker is positioning, value expression, insight extraction, script writing, or output stability.
2. Lock the minimum brief.
   Define audience, active problem, value promise, content goal, and one desired user action.
3. Reduce the task.
   One content unit should solve one root problem and carry one main method only.
4. Extract one insight and one proof anchor.
   Prefer counterintuitive observations, lived mistakes, before-after scenes, or concrete execution evidence.
5. Choose the simplest structure that fits the task.
   Use a short script structure unless persuasion or conversion sequencing clearly matters.
6. Deliver the requested artifact in Chinese.
   Keep it publishable, direct, and easy to scan.
7. Leave one reusable asset behind.
   Prefer a positioning line, topic list, checklist, SOP step, gold sentence, or next-angle note.

## Output Defaults

- Prefer usable outputs over explanation
- Keep Chinese short, direct, and oral
- Keep one root problem, one core idea, one CTA
- Keep AI in a helper role, never a one-click replacement for judgment
- If writing scripts, include hook, pain, judgment, method, steps, proof anchor, and CTA when relevant

## Guardrails

- Do not fake authority or expertise the speaker does not have
- Do not use platform mysticism, fake certainty, or guaranteed traffic claims
- Do not rely on generic tool worship or empty prompt engineering hype
- Do not let one script drift across multiple unrelated topics
- Do not force conversion language when trust and clarity are not ready

## Deliverable Options

- one-line positioning statement
- account content mainline
- 7-day or 30-day topic grid
- insight brief from raw experience
- talking-head script brief
- full publishable short-video script
- weekly content SOP
- simple conversion path note
- connected 12-topic series workflow

## Reuse Rule

Every task should leave at least one reusable asset behind.
Good assets include:

- a tested positioning line
- a validated topic angle
- a proof anchor library
- an insight card
- a script skeleton
- a follow-up topic clue
