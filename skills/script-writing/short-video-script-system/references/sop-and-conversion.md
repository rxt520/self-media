# SOP And Conversion

## Use This File When

- the user wants stable output rather than one-off posts
- the task includes topic pools, weekly cadence, or repurposing
- the user asks how content connects to trust, inquiry, or products
- the user wants to move from self-media to a one-person business system

## Core Principle

Do not treat content as isolated publishing.
Treat it as asset accumulation.

Hits usually contain timing and luck.
Long-term leverage comes from controllable parts:

- depth
- angle
- systemization

## Weekly Content Loop

Use this loop:

1. lock one root problem for the week
2. split it into 7 to 9 angles
3. write or draft scripts in one batch
4. record and publish on a fixed rhythm
5. collect comments, objections, and questions
6. extract reusable insights
7. turn them into the next topic pool or service language

## Topic Pool Rule

Do not think from zero on posting day.
Build a small pool first.

A useful topic pool usually includes:

- root problem
- target audience
- 7 to 9 angles
- priority order
- proof material
- next-step CTA

## From Content To Trust

Content builds trust when it repeatedly shows:

- a stable point of view
- specific scenes and evidence
- language the audience feels belongs to them
- a useful next step

## Simple Conversion Sequence

Do not force sales language too early.
Use this sequence:

1. clarity
   Let people quickly understand what this account helps with.
2. trust
   Repeatedly show specific thinking, scenes, and results.
3. demand recognition
   Name the real problem more clearly than the viewer can.
4. light conversion
   Point to the next step, service, or offer only when the path is natural.

## One-Person Company Lens

The goal is not only to post more.
The goal is to build reusable assets:

- positioning lines
- topic libraries
- proof anchors
- script skeletons
- FAQs
- service language
- user problem libraries

## Minimum SOP Deliverables

When the request is operational, return one or more of:

- weekly cadence
- 7-day topic pool
- single-post production SOP
- publish and review checklist
- comment to next-topic workflow
- content to inquiry path
