# Script Patterns

## Use This File When

- the user asks for a short-video script
- the user wants a hook, title, or rewrite
- the user has a topic but no usable structure
- the user wants a publishable talking-head draft

## North Star

Write content that can be understood fast, finished smoothly, remembered clearly, and acted on immediately.

Default script logic:

1. one-line insight
2. why it is true
3. how to do it

## Mandatory Gates

A usable script should pass all three:

1. active pain
   Hit a real problem the audience is feeling now.
2. executable action
   Give at least one concrete move the viewer can do alone.
3. memorable line
   Include one line worth saving or screenshotting.

If the topic cannot pass these gates, change the angle before writing the full draft.

## Core Script Rule

One script should have:

- one audience
- one root problem
- one core method
- one to three steps
- one proof anchor
- one CTA

## Recommended Structure

Use this talking-head sequence by default:

1. hook
2. pain scene
3. judgment
4. core method
5. steps
6. proof anchor
7. CTA

## Structure Selection

- Use `APA` for fast viewpoint reversal
- Use `PAS` for pain attack plus solution
- Use `AIDA` when persuasion or action sequencing matters more than teaching depth
- Use a simple checklist script when the user only needs direct execution guidance

## Hook Rules

Open immediately with conflict.
Useful patterns:

- 你不是不会做，是你一开始就做错了顺序
- 很多人以为卡在 X，其实真正拖住你的是 Y
- 最危险的不是 X，是 Y
- 不是 X 更重要，而是 Y 要先定

Avoid:

- greetings
- self introduction
- "今天分享 3 个方法"

## Mid-Section Rules

- move from pain to reframing
- move from reframing to method
- move from method to action
- do not loop back into theory after the steps

## Proof Anchor Rules

Every script needs at least one concrete anchor.
Prefer:

- time change
- quantity change
- before-after comparison
- concrete execution scene

If the proof is personal observation only, add a downgrade sentence.
Example:

`这是我自己的流程观察，不是所有账号都会一模一样。`

## AI Boundary

When the script uses AI, keep AI in a helper role.
Useful sentence:

`AI 辅助提效，最后怎么用、怎么改，还是你自己定。`

## CTA Rules

Use one CTA only.
Preferred order:

1. save
2. low-pressure comment
3. weak follow-up clue

If the goal includes follow reasons, say what comes next before the CTA.

## Output Bundle

When useful, return:

- title
- subtitle
- angle
- duration
- opening hook
- full script
- proof anchor
- AI boundary sentence
- CTA
