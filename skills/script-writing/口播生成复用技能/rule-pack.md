## Fixed Audience

- people who want to build self-media accounts
- they want real-person talking-head content, not digital humans
- they want AI to help with writing and editing efficiency
- they care about growth, follow reasons, and stable output

## Fixed Boundaries

- real-person talking-head only
- AI can help with draft writing, structure cleanup, and editing efficiency
- AI cannot replace human judgment
- do not write pure tool搬运, generic赚钱鸡汤, or exaggerated profit promises

## Preferred Angles

Use these first:

1. AI editing efficiency
2. account focus
3. stable output

Use carefully:

- hook structure
- AI writing

These two are easier to get play-feel, but harder to create a strong long-term follow reason.

## Script Requirements

- full version duration: 80 to 110 seconds
- main title: at most 10 Chinese characters
- title must have a subtitle
- one script solves only one root problem

## Seven-Part Structure

1. opening hook
2. pain scene
3. judgment
4. one core method
5. one to three steps
6. proof anchor
7. one CTA

## Opening Rules

The first sentence should start with conflict or judgment.

Preferred patterns:

- 你不是不会做，是你一开始就做错了顺序
- 很多人以为是 X，其实真正拖住你的是 Y
- 最危险的不是 X，是 Y

Do not start with:

- self introduction
- greeting
- “今天分享 3 个方法”

## Core Middle Rules

- only one root problem
- only one core method
- steps must be directly executable
- after steps, only allow one result-closing sentence plus one CTA
- do not return to explain a second layer of theory

## Proof Anchor Rules

Every script needs at least one hard proof anchor.

Preferred anchors:

- time change
- quantity change
- before and after comparison
- concrete execution scene

Examples:

- 连续两周只讲一条主线
- 最近 10 条内容先停掉不服务主线的
- 单条剪辑时间从接近 2 小时压到 40 分钟左右
- 新访客看主页前三屏更容易看懂账号在讲什么

If the proof is only personal observation, add a downgrade sentence:

`这是我自己的流程观察，不是所有账号都会一模一样。`

## AI Boundary Sentence

For AI-related scripts, add this fixed sentence after the steps:

`AI 辅助提效，最后怎么用、怎么改，还是你自己定。`

## Ending Rules

Keep only one CTA.

Preferred CTA order:

1. save
2. low-pressure comment
3. weak follow-up

Preferred ending formula:

- one statement-style next clue
- one save CTA

Example:

`这条线我后面继续往下拆。这条你先存着。`

## Banned Ending Patterns

- 你先关注
- 评论区扣 XX
- 我直接给你 XX
- 点赞关注收藏转发一条龙
- 私信我领资料

## Follow-Reason Rule

If the script wants to build follow reasons, the sentence before the CTA must state what comes next.

Prefer:

- 这条线我后面继续往下拆
- 下一条我继续讲这个问题
- 后面我接着拆它怎么落地

Avoid:

- 如果你想继续看
- 如果你还想看
