# Newbie AI Self-Media 12-Topic Workflow

## Use This File When

- the user already has a fixed list of AI-related self-media topics
- the user wants isolated topics turned into one connected series
- the user needs a publishing order, follow reason chain, or batch script brief
- the account is aimed at beginners learning self-media with AI assistance

## Fixed Account Fit

This workflow assumes the account promise is close to:

`一个自媒体新手，如何借助 AI，把不会做的内容慢慢做会。`

Audience:

- new creators
- people who want to use AI to improve content efficiency
- people stuck on topics, scripts, speaking, continuity, or review

## Series Goal

Do not treat these 12 topics as separate tips.
Treat them as one growth path:

1. help the viewer quickly understand the account mainline
2. prove the creator has a repeatable system
3. show practical AI use in real content work
4. build trust by showing boundaries, mistakes, and real observations

## Topic Map

Use the list below as the upstream series brief before generating full scripts.

### 1. 新手主页太散，怎么用 AI 帮你收主线

- stage: positioning
- root problem: profile feels random and hard to understand
- core method: one-line mainline, then screen topics against it
- proof anchor: recent posts stopped or filtered by mainline
- next clue: 主线定了，下一步不是硬拍，而是把一个主题拆成系列

### 2. 不知道拍什么系列，怎么让 AI 把一个主题拆成 10 条

- stage: series design
- root problem: one topic stays abstract and cannot sustain output
- core method: split one root problem into 10 repeatable sub-angles
- proof anchor: one topic becomes hooks, structure, pitfalls, review, and cases
- next clue: 系列有了，再继续往下拆成 7 天可直接拍的安排

### 3. 不会做选题的人，怎么让 AI 先帮你拆 7 天内容

- stage: topic planning
- root problem: topic choice starts from zero every day
- core method: lock one root problem and batch-plan 7 daily angles
- proof anchor: one-week grid prepared in one sitting
- next clue: 题有了，但很多人还是卡在第一版稿子

### 4. 不会写口播稿的人，怎么让 AI 先给你打初稿

- stage: drafting
- root problem: blank page and no first version
- core method: feed audience, problem, structure, and duration first, then let AI draft
- proof anchor: from empty page to editable first draft
- next clue: 有了初稿，不代表你已经讲顺了

### 5. 口播总是讲不顺，怎么用 AI 先整理表达顺序

- stage: delivery clarity
- root problem: content sounds messy even when the point is right
- core method: let AI reorder hook, pain, judgment, method, and steps
- proof anchor: the same idea becomes easier to say in one take
- next clue: 讲顺了之后，很多人又卡在后期改太久

### 6. 一条视频改太久，怎么让 AI 帮你先粗剪思路

- stage: editing efficiency
- root problem: editing starts from polish instead of usable version
- core method: rough cut first, then let AI assist with transcript and emphasis
- proof anchor: editing time compressed after switching sequence
- next clue: 但如果每次发前才想题，再会剪也发不稳

### 7. 每次发之前才想题，怎么用 AI 建一个题池

- stage: output stability
- root problem: posting depends on same-day inspiration
- core method: build a reusable topic pool before recording day
- proof anchor: topic pool supports multiple recording sessions
- next clue: 发出去之后，下一步不是凭感觉，而是先会复盘

### 8. 不会复盘数据，怎么让 AI 帮你整理问题

- stage: review
- root problem: data exists but the creator cannot diagnose what failed
- core method: organize each post by hook, structure, topic, CTA, and response signal
- proof anchor: review notes become next-round decisions
- next clue: 复盘不是终点，真正稳定的人会把学习也变成内容

### 9. 我每天怎么用 AI 学习并顺手变成一条内容

- stage: daily system
- root problem: learning and publishing feel like separate tasks
- core method: convert daily learning into one note, one insight, one script angle
- proof anchor: one study input becomes one visible content output
- next clue: 讲了这么多方法，也要讲清楚 AI 最容易被误会的地方

### 10. 新手最容易误会 AI 的 3 件事

- stage: cognition correction
- root problem: beginners expect AI to replace skill, taste, and judgment
- core method: correct the biggest false expectations
- proof anchor: real mistakes from early use
- next clue: 误会纠正完，再把边界讲透，信任才更稳

### 11. AI 能帮你提效，但不能替你判断的 3 件事

- stage: boundary and trust
- root problem: over-reliance on AI weakens creator judgment
- core method: separate helper tasks from human decisions
- proof anchor: examples where the creator still had to decide
- next clue: 最后回到真实过程，讲 AI 到底在哪些地方真帮到了我

### 12. 我做自媒体两个月，哪些地方是 AI 真帮到我的

- stage: lived proof
- root problem: viewers hear methods but still want real evidence
- core method: summarize actual gains in speed, clarity, output stability, and learning
- proof anchor: before-after workflow or time savings
- next clue: 这条线后面还能继续拆每一个具体环节

## Functional Role Split

Use these roles when planning the series:

- mainline-building topics: 1, 2, 3
- efficiency-proof topics: 4, 5, 6, 7, 8, 9
- trust-building topics: 10, 11, 12

## Handoff Package To Script Generator

Before sending this series to a lower-level script skill, lock:

- account one-line mainline
- final publishing order
- one root problem per topic
- one proof anchor per topic
- one next-topic clue per topic
- one CTA style for the whole series

Recommended CTA style:

- save first
- low-pressure follow-up clue second

Recommended closing formula:

`这条线我后面继续往下拆，这条你先存着。`

## Output Bundle

When this workflow is requested, return:

1. one-line account mainline
2. the 12-topic grouped roadmap
3. a per-topic brief with root problem, method, proof anchor, and next clue
4. optional lower-level generation handoff prompt
